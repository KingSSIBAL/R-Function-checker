Authentication and Private Repositories

Overview

The autograder uses secure authentication with GitHub tokens for private repository access. This ensures test cases and reference implementations are protected from unauthorized access.

Security Features

  • Private Repository Support: Test cases stored in private GitHub repositories
  • Encrypted Token: GitHub token encrypted in compiled binary
  • No Exposure: Students cannot extract credentials
  • Hidden Tests: Test details can be hidden from students

## Setting Up Secure Mode

The autograder provides easy-to-use functions for configuring secure mode.

### Quick Setup (Recommended)

Use the one-command setup function:


``` r
library(autograder)

# One-command configuration and build
autograder_configure(
  url = \"https://api.github.com/repos/YourOrg/PrivateRepo/contents\",
  token = \"github_pat_xxxxxxxxxxxxxxxxxxxx\",
  install = TRUE  # Install after building
)

Or use the interactive wizard:

autograder_setup_wizard()
# Guides you through each step with validation

Or use the RStudio Addin (Addins menu → Configure Autograder).

Manual Setup

If you prefer manual configuration:

Step 1: Create a Private Repository

  1. Create a new private repository on GitHub

  2. Add your test case files:

    private-repo/
├── functions/
│   ├── _problems.R
│   └── your_functions.R
└── data/
    └── your_data.csv

Step 2: Generate a GitHub Token

  1. Go to GitHub → Settings → Developer Settings → Personal Access Tokens
  2. Click Generate new token (fine-grained)
  3. Configure:
    • Name: autograder-course-name
    • Expiration: End of semester
    • Repository access: Only select repositories → your private repo
    • Permissions: Contents → Read-only
  4. Copy the generated token

Step 3: Configure the Autograder

Create a .env file in the tools/ directory:

# tools/.env
AUTH_MODE=secure
REPO_URL=https://api.github.com/repos/YourOrg/PrivateRepo/contents
GITHUB_TOKEN=github_pat_xxxxxxxxxxxxxxxxxxxx

Step 3: Test Your Token

# Verify token works before building
autograder_test_token(
  url = \"https://api.github.com/repos/YourOrg/PrivateRepo/contents\",
  token = \"github_pat_xxxxxxxxxxxxxxxxxxxx\"
)
#> ✓ Token is valid
#> ✓ Repository accessible

Step 4: Build the Package

# Option A: Use the built-in configure function
autograder_configure(
  url = \"https://api.github.com/repos/YourOrg/PrivateRepo/contents\",
  token = \"github_pat_xxxxxxxxxxxxxxxxxxxx\",
  install = TRUE
)

# Option B: Manual process using the helper script
source("tools/encrypt_url_helper.R")

# Verify token works
test_token()
#> ✓ Token is valid
#> ✓ Repository accessible
#> ✓ Functions folder found

# Configure secure mode
setup_secure_mode()
#> ✓ Secure mode configured
#> ✓ Token encrypted and embedded
#> ✓ Package ready to build

# Build the package
build_package()

Step 5: Distribute to Students

Distribute the built package (.tar.gz or .zip). Students install normally:

install.packages("autograder_0.4.0.tar.gz", repos = NULL, type = "source")
library(autograder)
autograder("problem_name")

The token is embedded and encrypted—students don’t need to configure anything.

Security Considerations

Token Security

  • Token is encrypted using AES-256 with a derived key
  • Each package build generates unique encryption
  • Token is never exposed in plain text
  • Students cannot extract the original token

Best Practices

  1. Use fine-grained tokens with minimal permissions
  2. Set expiration dates aligned with your course
  3. Rotate tokens each semester
  4. Scope to specific repository only
  5. Monitor repository access for suspicious activity

Checking Authentication Status

From R

library(autograder)

# Get authentication info (internal function)
info <- autograder:::.cpp_get_auth_info()
info$mode      # Always "secure"
info$has_token # TRUE if token configured

# Get auth mode directly
autograder:::.cpp_get_auth_mode()
#> [1] "secure"

From C++

The package exposes C++ functions for authentication:

// In C++ extensions (internal)
std::string mode = get_auth_mode();     // Always "secure"
std::string token = get_github_token(); // encrypted token

Troubleshooting

“Token validation failed”

Error: GitHub token validation failed

Solutions: - Verify token hasn’t expired - Check token has Contents: Read permission - Confirm repository name is correct - Ensure repository is accessible with the token

“Repository not found”

Error: Repository not accessible

Solutions: - Verify the REPO_URL format: - For API: https://api.github.com/repos/OWNER/REPO/contents - For raw: https://raw.githubusercontent.com/OWNER/REPO/BRANCH/repo - Check repository exists and is accessible

“Function not found” (but exists in repo)

Error: Function 'problem_name' not found

Solutions: - Ensure functions are in functions/ subfolder - Verify _problems.R lists the function - Check file permissions in repository
